```text
  Bottlenecks are for understanding; lack of bottlenecks is for creating.
  If you want to compress a signal (Encoder),
  use a bottleneck If you want to generate a high-fidelity signal (Generator),
  keep the architecture "wide" and "spatially aware" so information is never choked.
```
